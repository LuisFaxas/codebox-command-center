**public/hooks-install.html** — Create a self-contained HTML page with:

1. **Platform detection**: Read `navigator.userAgent` to auto-detect Windows/macOS/Linux and pre-select the matching tab.

2. **Three tabs**: macOS, Windows, Linux — each with platform-specific instructions.

3. **Step 1 — Download the hook script**: Show the `curl` (macOS/Linux) or `Invoke-WebRequest` (Windows) command with the full server URL baked in. The URL should use `window.location.origin` so it adapts to however the user is accessing the page (LAN IP, Tailscale IP, etc.):
   - macOS: `curl -o ~/.claude/hooks/notify-trigger.cjs {origin}/hooks/script`
   - Windows: `Invoke-WebRequest -Uri "{origin}/hooks/script" -OutFile "$env:USERPROFILE\.claude\hooks\notify-trigger.cjs"`
   - Linux: `curl -o ~/.claude/hooks/notify-trigger.cjs {origin}/hooks/script`

4. **Step 2 — Add hooks to settings.json**: Show the exact JSON snippet to merge into `~/.claude/settings.json`. Use the server's origin URL in the `VOICE_NOTIFY_URL` env var within the hook command. The snippet must include all 4 hook types (Stop, Notification, PostToolUse, SessionStart) with:
   - macOS/Linux paths: `~/.claude/hooks/notify-trigger.cjs`
   - Windows paths: use `%USERPROFILE%\\.claude\\hooks\\notify-trigger.cjs` (escaped backslashes in JSON)
   - Each command prefixed with `VOICE_NOTIFY_URL={origin}` env var
   - Timeout: 5 for Stop/Notification, 3 for PostToolUse/SessionStart

5. **Step 3 — Test**: Show a test command that pipes JSON to the hook script and a "Test Connection" button that POSTs to `/hooks/test` from the browser.

6. **Styling**: Use the same FAXAS brand CSS variables as the main dashboard. Inline the essential CSS (dark theme, `--bg-primary`, `--text-primary`, `--accent` etc.) so the page is self-contained. Use `<pre>` blocks with a "Copy" button for each command block.

7. **Prerequisites check**: Show a note at the top: "Requires Node.js installed on the target machine. Verify with `node --version`."

8. Per D-08: Add a clear warning that these instructions are for the REMOTE machine only and will not affect CodeBox's configuration.

**server.js** — Add three routes (before the static file fallback):

1. `GET /hooks/install` — Serve the `public/hooks-install.html` file:
   ```javascript
   } else if (pathname === '/hooks/install') {
     const installPath = join(import.meta.dirname, 'public', 'hooks-install.html');
     try {
       const data = readFileSync(installPath, 'utf8');
       res.writeHead(200, {'Content-Type': 'text/html'});
       res.end(data);
     } catch(e) {
       res.writeHead(404); res.end('Not found');
     }
   ```

2. `GET /hooks/script` — Serve the raw `hooks/notify-trigger.cjs` file for download:
   ```javascript
   } else if (pathname === '/hooks/script') {
     const scriptPath = join(import.meta.dirname, 'hooks', 'notify-trigger.cjs');
     try {
       const data = readFileSync(scriptPath);
       res.writeHead(200, {
         'Content-Type': 'application/javascript',
         'Content-Disposition': 'attachment; filename="notify-trigger.cjs"'
       });
       res.end(data);
     } catch(e) {
       res.writeHead(404); res.end('Not found');
     }
   ```

3. `POST /hooks/test` — Accept a test ping and return success with the server's hostname:

   ```javascript
   } else if (pathname === '/hooks/test' && req.method === 'POST') {

   ```
