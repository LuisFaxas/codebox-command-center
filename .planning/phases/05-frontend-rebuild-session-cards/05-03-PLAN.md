    **public/modules/sdk-client.js** -- Fetch wrapper for SDK proxy endpoints:
    - `fetchSdkSessions()` -- GET /sdk/sessions, returns array or empty array on error
    - `fetchMessages(sessionId, limit = 20, offset = 0)` -- GET /sdk/sessions/${sessionId}/messages?limit=${limit}&offset=${offset}, returns array or empty array on error
    - `sendSdkResponse(sessionId, text)` -- POST /sdk/sessions/${sessionId}/send with JSON body {text}, returns response JSON. On error returns `{ ok: false, error: 'Network error' }`.
    - All functions use async/await with try/catch. No polling -- these are called on demand.

    **public/modules/sessions.js** -- Session card rendering:

    Export `initSessions()` function called from app.js. It:
    1. Gets reference to `#session-grid` element
    2. Subscribes to `session:update` -- calls `renderCard(sessionData)`
    3. Subscribes to `session:remove` -- removes card element by sessionId
    4. Subscribes to `session:select` -- updates selected visual state

    `renderCard(session)` function:
    - Creates or updates a card element with `data-session-id` attribute
    - Card class: `.session-card .glass-subtle` (switches to `.glass-card` when selected)
    - Card click handler: calls `setSelectedSession(session.sessionId)`
    - Card contents per UI-SPEC Component 2:
      - Top row: project name (h3, `.text-heading`, truncate with ellipsis) + status badge (span with `.status-badge .status-{status}`)
      - Status badge: pill shape. Background uses status color at 15% opacity. Text is full status color. Content = `statusLabel(session.status)` (uppercase). CSS classes: `.status-badge`, with modifier `.status-done`, `.status-attention`, `.status-working`, `.status-stale`
      - Second row: machine name (span, `.text-label`, secondary color) with small inline SVG device icon (16x16, just a simple monitor outline)
      - Third row: last message preview (p, `.text-body`, monospace font, muted color, 1 line truncated). Content = `session.questionText` if attention status, or "Completed" for done, "Working..." for working, "Inactive" for stale. Use `escapeHtml()`.
      - Bottom row: duration (span, `.text-label`, muted, right-aligned). Content = `formatRelativeTime(session.lastActivity)`. If stale, show "Inactive" in muted text instead.
      - Reply button: ONLY if status === 'attention'. Pill button with accent gradient background, white text, label "Send Reply". Click handler: `setSelectedSession(session.sessionId)` (same as card click -- focuses conversation panel).
    - Card with attention status: add `.status-attention` class for pulsing left border animation
    - Card with selected state: replace `.glass-subtle` with `.glass-card`, add `.selected` class for accent border and glow
    - Cards are cursor: pointer
    - Re-render on every `session:update` event (simple approach: find existing card by data-session-id, update innerHTML)

    Update duration display: start a `setInterval(60000)` that updates all duration displays.
  </action>
