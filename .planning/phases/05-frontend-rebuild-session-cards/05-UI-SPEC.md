## Spacing Scale

| Token | CSS Variable | Value | Usage |

| xl | `--space-xl` | 24px | Panel padding, sidebar internal padding |

- Touch target minimum: 44px -- interactive buttons, reply input submit

## Typography

| Role | Size | Weight | Line Height | Font Family | CSS Variable |
|------|------|--------|-------------|-------------|-------------|
| Label | 12px (0.75rem) | 600 (semibold) | 1.4 | DM Sans | `--text-label` |
| Body / Code | 14px (0.875rem) | 400 (regular) | 1.5 | DM Sans / JetBrains Mono | `--text-body` |
| Heading | 18px (1.125rem) | 600 (semibold) | 1.3 | DM Sans | `--text-heading` |
| Display / Metric | 24px (1.5rem) | 600 (semibold) | 1.2 | DM Sans | `--text-display` |

Usage mapping:
- **Label (12px, 600):** Machine name, timestamps, connection label, status badge text, SDK disclaimer, section headings in sidebar, voice selector labels, slider labels, uppercase header title
- **Body (14px, 400):** Body text, voice selector buttons, template textarea, tab switcher text, generate button text
- **Body with JetBrains Mono (14px, 400):** Conversation messages, code blocks, last message preview, response input. Use `--text-body` size with mono font family.
- **Heading (18px, 600):** Project name on cards, panel header project name, sidebar section headings
- **Display (24px, 600):** Metric values (session count, etc.), page-level headings if needed

Notes:
- `font-display: swap` on Google Fonts link for fallback before load
- System font fallback stack: `'DM Sans', -apple-system, BlinkMacSystemFont, sans-serif`
- Mono fallback stack: `'JetBrains Mono', 'Fira Code', monospace`
- Metric values use `font-variant-numeric: tabular-nums` (from HUB-DESIGN.md pattern)
- Source: D-11 from CONTEXT.md, PORTFOLIO-DESIGN.md, HUB-DESIGN.md

## Glass System

| Tier | CSS Class | Background | Blur | Border | Radius | Usage |
|------|-----------|------------|------|--------|--------|-------|
| Subtle | `.glass-subtle` | `rgba(255,255,255,0.03)` | none | `1px solid var(--border-thin)` | 12px | Unselected session cards at rest |
| Medium | `.glass-card` | `rgba(255,255,255,0.05)` | `blur(16px)` | `1px solid var(--border-standard)` | 12px | Selected session card, conversation panel |
| Elevated | `.glass-panel` | `rgba(255,255,255,0.06)` | `blur(10px)` | `1px solid var(--border-standard)` | 12px | Sidebar, header |

Performance rule: Apply `backdrop-filter: blur()` only to the selected card, conversation panel, sidebar, and header. All other session cards use `.glass-subtle` (no blur) to avoid GPU overload with 10+ cards. Source: RESEARCH.md Pitfall 5.

## Interaction Contracts

### Card Selection
- Click any session card to select it
- Selected card gets `.glass-card` treatment + accent border
- Previously selected card reverts to `.glass-subtle`
- Conversation panel loads messages for selected session via `GET /sdk/sessions/:id/messages`
- Only one card selected at a time

### Response Input
- Visible only when selected session has status "attention"
- `Enter` key submits (without Shift). `Shift+Enter` inserts newline.
- Submit button disabled while sending (shows spinner or opacity 0.5)
- On success: input clears, new message appears in conversation list
- On failure: error toast appears, input content preserved

### Sidebar Toggle
- Click toggle button in header to collapse/expand sidebar
- Button has `aria-label="Toggle voice settings sidebar"`
- Collapsed: 48px wide, icon-only
- Expanded: 340px wide, full config UI
- State persisted in `localStorage` key `sidebar-collapsed`

### Toast Behavior
- Toasts stack vertically in `#toast-container` with 8px gap
- Maximum 5 visible toasts; oldest dismissed if exceeded
- Click toast to dismiss early
- Toasts do not block interaction with cards or conversation

### Session Card Attention Pulse
- Cards with "attention" status have a pulsing left border
- Pulse animation runs continuously until status changes
- If attention card is selected, pulse stops (user is actively viewing)

## Animation Contract

All animations use CSS only (no JS animation libraries). Source: D-18, Claude's Discretion.

| Animation | Property | Duration | Easing | Trigger |
|-----------|----------|----------|--------|---------|
| Card hover lift | `transform: translateY(-2px)` | 0.2s | ease | `:hover` |
| Card border transition | `border-color` | 0.2s | ease | select/deselect |
| Sidebar collapse | `width`, `opacity` | 0.3s, 0.2s | ease | toggle click |
| Toast enter | `transform: translateX(120%) -> 0` | 0.3s | ease-out | SSE notification event |
| Toast exit | `transform: 0 -> translateX(120%)` | 0.2s | ease-in | auto-dismiss or click |
| Attention pulse | `border-left-color` opacity cycle | 2s | ease-in-out | status === attention, infinite |
| Connection dot glow | `box-shadow` | 0.3s | ease | SSE connect/disconnect |

Reduced motion: wrap all animations in `@media (prefers-reduced-motion: no-preference)`. When reduced motion preferred, disable all transforms and use instant state changes.

## File Architecture (Claude's Discretion Decision)

ES modules with import maps. Source: RESEARCH.md recommended structure.

```
public/
  index.html              # HTML shell, CSS custom properties, import map, layout structure
  css/
    tokens.css            # All CSS custom properties from this spec
    layout.css            # Grid layout, responsive, sidebar
    components.css        # Glass cards, conversation, toast, sidebar components
  modules/
    app.js                # Entry point, bootstraps all modules
    state.js              # Shared state (sessions Map, selectedSessionId, config)
    sse.js                # SSE connection, event routing
    sessions.js           # Session card rendering, grid management
    conversation.js       # Conversation panel, message rendering, response input
    sidebar.js            # Config sidebar (voice, template, collapse)
    toasts.js             # Toast notification system
    audio.js              # Voice playback (existing logic extracted)
    sdk-client.js         # Fetch wrapper for /sdk/* endpoints
    utils.js              # escapeHtml, formatDuration, formatRelativeTime
```
