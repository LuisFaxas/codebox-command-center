    **public/modules/toasts.js** -- Toast notification system:

    Export `initToasts()` and `showToast(type, project)`.

    `initToasts()`:
    - Gets reference to `#toast-container` (MUST already exist in index.html)
    - Subscribes to `trigger` event from state dispatch: calls `showToast(data.type, data.project)`

    `showToast(type, project)`:
    - Creates a div with class `toast` (CRITICAL: Playwright tests look for `.toast` inside `#toast-container`)
    - Add class `toast-${type}` for done (green left border) or question (amber left border)
    - Toast content:
      - Small inline SVG icon: checkmark for done (green), question mark for question (amber). Keep SVG simple, ~20x20.
      - Text: `"{project}" is done` for done type, `"{project}" needs attention` for question type. Use `escapeHtml(project)`.
    - Styling: `.glass-card` background, padding 16px, max-width 360px, 12px radius
    - Left border: 3px solid `var(--status-done)` for done, `var(--status-attention)` for question
    - Animation: add a class `.toast-enter` that triggers CSS `translateX(120%) -> translateX(0)` over 0.3s ease-out
    - Auto-dismiss: setTimeout to remove after 8000ms for done, 15000ms for question (per NOTIF-03)
    - Dismiss animation: add `.toast-exit` class (translateX(0) -> translateX(120%), 0.2s ease-in), then remove from DOM after animation
    - Click to dismiss early: click handler on toast
    - Max 5 toasts visible: if >5 in container, remove oldest
    - Stack with 8px gap between toasts

    **public/modules/audio.js** -- Voice notification playback:

    Export `initAudio()`.

    `initAudio()`:
    - Creates a hidden `<audio>` element
    - Subscribes to `trigger` event from state dispatch
    - On trigger: set audio src to `/notify-wav?type=${type}&project=${encodeURIComponent(project)}&t=${Date.now()}` (cache-bust)
    - Call `audio.play()`. Wrap in try/catch -- browsers may block autoplay until user interaction. If blocked, silently fail (voice is best-effort; push notifications are the fallback).
    - No polling. Only plays when trigger event arrives via SSE.

    Extracted from current index.html logic. The existing monolith uses a similar pattern with fetch to /notify-wav and Audio API.
  </action>
