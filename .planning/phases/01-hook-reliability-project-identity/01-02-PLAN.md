Test 12 -- Debounce behavior note for documentation:
The 3-second debounce window per (type, project, sessionId) is a locked decision (D-10, D-11). If two legitimate responses complete within 3 seconds in the same session, the second will be suppressed. This is an intentional tradeoff: sub-agent noise prevention is prioritized over rare rapid legitimate events. Document this in the SUMMARY.

If any test fails, diagnose and fix the issue in the relevant file (hooks/notify-trigger.js or server.js).
