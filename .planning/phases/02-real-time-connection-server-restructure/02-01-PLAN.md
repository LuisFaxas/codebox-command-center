Create `sse.js` — the SSE event bus module. Per D-01, D-02, D-03, D-08.

Imports: none needed (pure Node.js, no external deps).

Module-level state:
- `const clients = new Set()` — active SSE response objects
- `const buffer = []` — circular replay buffer
- `const BUFFER_SIZE = 100` — max events in buffer (Claude's discretion)
- `let eventId = 0` — monotonically incrementing event ID

**`formatSSE(event)` (internal function)**
Takes `{ id, type, data }` object, returns SSE-formatted string:
```
id: ${event.id}\nevent: ${event.type}\ndata: ${JSON.stringify(event.data)}\n\n
```
CRITICAL: Must end with `\n\n` (double newline). Single `\n` is a field separator, not a message terminator.

**`export function emit(eventType, data)`**
Per D-02, supports four event types: `trigger`, `session:alive`, `config:updated`, `connection:health`.
1. Increment `eventId`
2. Create event object: `{ id: eventId, type: eventType, data, timestamp: Date.now() }`
3. Push to buffer; if `buffer.length > BUFFER_SIZE`, call `buffer.shift()` to remove oldest
4. Format with `formatSSE(event)`
5. Iterate `clients` Set, call `client.write(formatted)` for each

**`export function addClient(req, res, lastEventId)`**
Per D-01, single `/events` endpoint. Per D-03, replay missed events.
1. Write response headers:
   - `Content-Type: text/event-stream`
   - `Cache-Control: no-cache`
   - `Connection: keep-alive`
   - `X-Accel-Buffering: no`
   Status code: 200.
2. If `lastEventId` is truthy (not null/undefined/empty):
   - Parse as integer: `const lastId = Number(lastEventId)`
   - Find first event in buffer with `id > lastId`
   - Write all events from that index forward using `formatSSE()`
3. Add `res` to `clients` Set
4. Listen for `req.on('close', () => clients.delete(res))`

**`export function getClientCount()`**
Returns `clients.size`.

**Keepalive interval** — Per D-08, every 15 seconds:
```javascript
setInterval(() => {
  for (const client of clients) {
    client.write(': keepalive\n\n');
  }
}, 15000);
```
The `: keepalive\n\n` is an SSE comment (colon prefix) — browsers ignore it but it keeps the TCP connection alive through Caddy.
