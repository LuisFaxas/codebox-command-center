**Replace with EventSource connection:**
```javascript
// SSE connection — replaces polling (per D-01, D-07)
let eventSource = null;

function connectSSE() {
  eventSource = new EventSource('/events');

  eventSource.addEventListener('trigger', (e) => {
    const data = JSON.parse(e.data);
    count++;
    if (audio) { audio.pause(); audio = null; }
    audio = new Audio('/notify-wav?type=' + encodeURIComponent(data.type || 'done') + '&project=' + encodeURIComponent(data.project || '') + '&t=' + Date.now());
    audio.play();
    const label = data.type === 'question' ? 'Question' : 'Done';
    const proj = data.project ? ' (' + data.project + ')' : '';
    document.getElementById('status').textContent = label + proj + ' — ' + new Date().toLocaleTimeString();
    document.getElementById('count').textContent = count + ' notifications played';
    setTimeout(function() {
      document.getElementById('status').textContent = 'Listening for notifications...';
    }, 4000);
  });

  eventSource.addEventListener('config:updated', (e) => {
    serverConfig = JSON.parse(e.data);
    applyTab();
  });

  eventSource.addEventListener('connection:health', (e) => {
    // Server keepalive — connection is healthy
  });

  eventSource.onopen = () => {
    document.getElementById('status').textContent = 'Connected — listening for notifications...';
  };

  eventSource.onerror = () => {
    // EventSource automatically reconnects with backoff
    // Show reconnecting state
    document.getElementById('status').textContent = 'Reconnecting...';
  };
}
```

At the bottom of the script (where `loadConfig()` and `poll()` were), replace with:
```javascript
loadConfig();
connectSSE();
```

Keep all existing functions unchanged: `loadConfig()`, `switchTab()`, `applyTab()`, `generate()`, `loadVoices()`, `playVoice()`, `selectVoice()`.

Per D-09, Caddy config update is part of this phase.

Append the following site block to `/etc/caddy/Caddyfile` (after existing site blocks). Per D-08 and Research Caddy section:

```
voice-notifications.codebox.local {
    tls internal
    reverse_proxy localhost:3099 {
        flush_interval -1
        transport http {
            read_timeout 0
        }
    }
}
```

Key settings:
- `tls internal` — Self-signed cert for HTTPS (same pattern as other codebox.local sites in the Caddyfile)
- `flush_interval -1` — Disables response buffering, required for SSE instant delivery. Without this, Caddy may batch SSE events.
- `read_timeout 0` — No timeout on reading the upstream response, prevents Caddy from closing long-lived SSE connections.

After editing the Caddyfile, reload Caddy:
```bash
sudo systemctl reload caddy
```

Verify Caddy accepted the config:
```bash
sudo systemctl status caddy
```

Do NOT add `encode` (gzip/zstd) to this site block — encoding can interfere with SSE streaming (Research Pitfall 2).
