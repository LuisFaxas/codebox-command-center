Sections to implement:

**// === STATE ===**
```javascript
const sessions = new Map(); // key: sessionId, value: { project, machine, status, lastActivity, startedAt }
const activityFeed = [];     // array of { type, project, machine, timestamp }, newest first, max 50
let notifyCount = 0;
```

**// === SSE CONNECTION ===**
- `connectSSE()`: Create EventSource to /events
- `trigger` listener: Update session in Map (status = type === 'question' ? 'attention' : 'done'), add to activityFeed (unshift, cap at 50), call `showToast(data.type, data.project)`, call `playVoice(data.type, data.project)`, increment notifyCount, call `renderSessions()` and `renderFeed()`
- `session:alive` listener: Update session (status = 'working', lastActivity = Date.now(), preserve startedAt), call `renderSessions()`
- `config:updated` listener: Store config locally
- onopen: Update header connection dot to green
- onerror: Update header connection dot to red, show "Reconnecting..." text. If CLOSED state, setTimeout(connectSSE, 3000)

**// === SESSION GRID ===**
- `renderSessions()`: Clear session-grid container, iterate sessions Map, create session card elements:
  - Card has data-status attribute for CSS styling
  - Project name (h3, truncated with ellipsis if long)
  - Status badge: "Working" (blue), "Done" (green), "Needs Attention" (amber) — with colored dot
  - Machine name (small, muted text)
  - Last activity: relative time ("2s ago", "5m ago") using `formatRelativeTime(timestamp)`
  - Duration: calculated from startedAt to now, formatted as "Xm" or "Xh Ym"
  - Sort: attention first, then working, then done, then stale

**// === SESSION TTL ===** (per D-07 discretion)
- Every 10 seconds, run TTL check:
  - Sessions with lastActivity > 5 minutes ago: set status to 'stale'
  - Sessions with lastActivity > 30 minutes ago: delete from Map
  - Call `renderSessions()` if any changes

**// === ACTIVITY FEED ===**
- `renderFeed()`: Clear feed container, render activityFeed array as feed-items
  - Each item: icon (checkmark for done, "?" for question), project name (bold), event type, timestamp (HH:MM:SS format)
  - Color-code by type (green for done, amber for question)

**// === TOAST SYSTEM ===** (per discretion: top-right, stack down, max 5)
- `showToast(type, project)`: Create toast DOM element with:
  - Icon: checkmark for done, "?" for question
  - Label: "Done" or "Needs Attention"
  - Project name
  - Timestamp
  - Left border color: green (done) or amber (question)
  - Prepend to #toast-container
  - If > 5 toasts, remove oldest
  - Auto-dismiss: 8s for done, 15s for question (per research recommendation)
  - Dismiss animation: opacity transition to 0, then remove element after 300ms

**// === VOICE PLAYBACK ===**
- `playVoice(type, project)`: Create Audio element pointing to `/notify-wav?type=${type}&project=${encodeURIComponent(project)}&t=${Date.now()}`. Call audio.play(). Handle play() rejection silently (autoplay policy).

**// === UTILITY ===**
- `formatRelativeTime(ts)`: Returns "Xs ago", "Xm ago", "Xh ago" based on elapsed time
- `formatDuration(startTs)`: Returns "Xm" or "Xh Ym" based on elapsed

**// === INIT ===**
- Call `connectSSE()` on page load
- Start TTL interval
