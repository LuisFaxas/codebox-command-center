**3. Add JavaScript section `// === PUSH NOTIFICATIONS ===`:**

`urlBase64ToUint8Array(base64String)`: Convert VAPID public key from base64url to Uint8Array (use exact implementation from research).

`async function subscribePush()`:
1. Check `'serviceWorker' in navigator && 'PushManager' in window` — if not supported, return null
2. Register service worker: `await navigator.serviceWorker.register('/sw.js')`
3. Wait for ready: `await navigator.serviceWorker.ready`
4. Fetch VAPID key: `const { publicKey } = await (await fetch('/vapid-public-key')).json()`
5. Check existing subscription: `const existing = await reg.pushManager.getSubscription()`
6. If existing, send to server anyway (re-subscribe pattern from Pitfall 3), return existing
7. If no existing: `const sub = await reg.pushManager.subscribe({ userVisibleOnly: true, applicationServerKey: urlBase64ToUint8Array(publicKey) })`
8. Send subscription to server: `fetch('/subscribe', { method: 'POST', headers: {'Content-Type': 'application/json'}, body: JSON.stringify(sub) })`
9. Return sub

`async function enablePush()`:
1. `const permission = await Notification.requestPermission()`
2. If permission === 'granted': call `subscribePush()`, hide banner, update connection status to show "Push enabled"
3. If permission === 'denied': hide banner (per D-04, never block — voice + toast still work)
4. If permission === 'default': hide banner (user dismissed browser prompt)

`function dismissBanner()`: Hide push-banner, set localStorage flag `push-banner-dismissed=true`

`async function initPush()`:
1. If Push API not supported, return (no banner)
2. If `Notification.permission === 'granted'`: silently call `subscribePush()` (re-subscribe for freshness)
3. If `Notification.permission === 'denied'`: do nothing (respect user choice)
4. If `Notification.permission === 'default'` AND localStorage `push-banner-dismissed` is not set: show push-banner

**4. Update `// === INIT ===` section:**
